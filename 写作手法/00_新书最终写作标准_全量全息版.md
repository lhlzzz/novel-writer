## 3. 最高硬门槛

任何章节只要命中以下任意一条，必须返修，不得进入 `我欠三百万，修bug续命/正文/`：

1. 前 300 字没有事故、冲突、倒计时、异常现场或不可回避的问题。
2. 前 300 字钩子低于 12/15。
3. 章末追读低于 12/15。
4. 没有新变量。
5. 没有战利品。
6. 没有代价账单。
7. 没有系统/世界架构推进。
8. 主角没有亲自动作或选择。
9. 只是重复追杀、逃跑、解释、查线索。
10. 正文一句一行、短句堆砌、像提纲或脚本。
11. 情绪爆点全用句号压平。
12. 人物功能化，像工具人。
13. 系统提示写成说明书，压过人物。
14. 章末只抛设定，没有戳到人物在乎的东西。
15. 前两章读完仍说不清主线：主角要什么、谁/什么在阻碍、失败代价是什么、下一步必须解决什么。
16. 没有 100-200 字主线简介，或简介只堆设定/氛围/世界观，不能让读者知道这本书到底看什么。
17. 环境、氛围、背景连续独立铺开，超过 3 句仍没有挂到冲突、动作、选择、危险、反差或伏笔上。
18. 用 `！？`、`……`、`。。。。`、口语词和破句伪装真人感，但段落本身仍在解释、铺垫、总结或绕开主题。
19. 任一段删掉后不影响主线、人物选择、规则推进、战利品、代价、关系刺点或伏笔，却仍被保留。
20. 总分低于 90 分。

### 3.1 读者反馈新增硬门禁：主线、描写、AI 味同源处理

2026-05-27 用户反馈已并入总标准：读者看了两章仍不明白主线；无关紧要描述太多，环境铺垫过多，半天读不到主题；AI 味重，读起来不像人写的；还需要一版能介绍主线的简介，否则读者看半天不知道在看什么。

这四条不是分散问题，统一按“主线不够显性、信息没有付费”处理：每句文字都必须为主线、冲突、人物、规则、代价、战利品、关系或伏笔付费。

#### 前两章主线可复述门禁

前两章读完，读者必须能用一句话复述：

```text
主角想要：______
阻碍是：______
失败代价是：______
所以接下来必须：______
```

正式开书前还必须写一版主线简介，用 100-200 字直接告诉读者这本书看什么：主角是谁、核心危机是什么、核心能力/规则是什么、长期对抗是什么、最大情感代价是什么。简介不能只堆设定、氛围、世界观和“命运齿轮开始转动”。

如果四格主线或简介说不清，先重构主线，不做表层润色。

第 1 章必须让读者知道：主角是谁、当前处境是什么、他想要什么、最大的麻烦是什么。第 2 章必须承接并升级这个麻烦，不能靠新设定、新人物、新环境稀释主线。

#### 描写付费门禁

环境描写只能嵌在人物行动、冲突推进、选择压力、危险限制、反差揭示或伏笔埋设里。禁止先独立铺环境，再慢慢进入主题。

每个环境句必须至少服务一项：

- 让主角更难。
- 暴露人物处境或性格。
- 制造选择压力。
- 埋下后文可回收证据。
- 形成反差或危险限制。
- 推动动作链继续发生。

都不服务，就删、压缩或后置。

#### 去 AI 味优先级门禁

去 AI 味不能先调标点、口语和破句。先删解释、删重复、删无功能铺垫，再处理句子自然度。

禁止把以下表层动作当成真人感：

- 硬塞 `！？`、`……`、`。。。。`。
- 为每段补口语词、语气词或破句。
- 用“像人说话”掩盖段落没有剧情功能。
- 用细节堆砌替代冲突推进。

验收顺序固定为：主线清晰 → 描写付费 → 人物行动 → 代价/战利品 → 文本自然度 → 标点口语。

#### 2026-05-29 读者正反馈与新增门禁：第001章爆相、回读爽感和失忆钩子

最新读者反馈确认：当前前三章方向成立，尤其第001章有爆相；“账”的逼格成立，小女孩事件选择有效，前三章整体不赖。后续返修不要推倒重来，要保护已成立的“账感”、羞耻开局、救人小事件和王海/苏念关系钩子。

关键修正：第001章第一次惩罚不需要写成说明书，也不要求读者第一遍完全看懂。读者读到第002-003章后倒回第001章，才意识到“原来那一下已经扣过代价”，这是好的回读爽感。执行标准应是首读有刺、回读恍然：首读能感到账给了七天但拿走了某个很私人的东西；回读时能清楚对上苏念记忆、王海查无此人和旧项目关系链抵押。禁止为了显性而写成规则解释或作者替读者划重点。

失忆机制新增执行口径：可高层参考《治愈游戏》《怪诞设计师》这类“主角不知道自己失忆，但读者知道”的钩子机制，只学高层叙事效果，不借用具体设定、桥段、角色或谜题。失忆不是单纯缺信息，而是读者比主角更早看见缺口；每次扣除都要让读者意识到“主角少了一块”，主角本人却只能从物件、旁人反应和账单痕迹里反推自己失去了什么。

## 4. 文本写法硬标准

### 4.1 不是“像真人”，而是当作家写

写作时不是做“去 AI 感”表层修饰，而是把自己当成一个活生生、有血有肉、对读者负责的作家。

正文必须让读者相信人物真实存在，让读者能共情、能心疼、能着急、能生气、能站队、能催更。

每个关键人物都要有：

- 具体欲望
- 羞耻或恐惧
- 软肋
- 执念
- 口癖或说话习惯
- 动作习惯
- 情绪压法
- 关系温度
- 后期可揭开的钩子

### 4.2 自然段标准

正文必须用真正的自然段承载动作、情绪、环境、心理和信息推进。

禁止：

- 一句一行
- 短句堆砌
- 试写碎片
- 提纲式正文
- 每个念头都单独断行
- 段落之间硬切

每个自然段应尽量包含连续的动作链、感受、环境、因果推进或信息变化。对话可以短，但叙述段必须有段落密度和流动感。

尤其在情绪戏、追逐戏、回忆戏、系统代价结算戏里，不能把一整段情绪切碎成句子清单。

### 4.3 情绪标点标准

正文不能全篇过度克制。惊讶、失控、暴怒、崩溃、反转爆点处，要自然使用：

- `！`
- `？！`
- `——`
- `……`
- 反问
- 重复
- 停顿
- 长短句变化

标点不是装饰，必须服务人物情绪和场景压力。不能只是形式上加几个标点，也不能滥用到像网文吼叫体。

### 4.4 场景质感标准

能用场景表达，就不要直接解释。

优先用这些东西承载情绪和信息：

- 砸门声
- 手机震动
- 冷掉的咖啡
- 空掉的相框
- B 超单
- 雨水
- 借条
- 旧钢笔
- 监控录像
- 门牌号错位
- 系统错误提示
- 某个人下意识的动作

不要直接写“他很痛苦”“她很失望”“这件事很重要”，要让读者从动作、物件、环境和对话里读出来。

## 5. 标题、简介和前 300 字

### 5.1 标题原则

标题必须一眼给出冲突、反差、异常任务或迫近危机。

可用结构：

1. 身份错位型：`我不是 X，我是 Y`
2. 地点反差型：`我在日常地点做超常动作`
3. 时限/终局型：`N 天 / N 小时 / N 次机会 + 终局危机`
4. 身份 + 异常任务型：`普通身份 + 反常动作 + 高压对象`
5. 核心动作 + 情绪强化型：`修 bug 续命 / 拒绝修复 / 让回收进程失控`
6. 代价收益型：`用记忆换寿命 / 修复成功，她忘了我`

章节标题不能只概括剧情，必须有点击欲。

### 5.2 前 300 字原则

前 300 字必须先把读者推入事故现场，再补设定。

至少包含以下一种线：

- 失控线
- 死亡线
- 账目线
- 关系线
- 规则线
- 舆论线
- 倒计时线

开篇不能慢慢解释世界观、职业背景、系统来源。主角必须立刻面临压力、选择或动作。

## 7. 前三章、前十章和长篇推进

### 7.1 前三章锁定

前三章必须完成：入局 → 规则 → 锁死。

1. 第 1 章：事故出现，主角被迫入局。
2. 第 2 章：表层答案被推翻，核心规则/能力/方法首次生效。
3. 第 3 章：长期代价出现，主角退路被堵死。

禁忌：

- 连续三章解释设定。
- 主角连续三章只观察不行动。
- 第 3 章还没有长期压力线。

### 7.2 前十章升级链

前十章每 1-2 章必须翻出一个更大答案，不能靠同一个悬念拖十章。

建议节奏：

- 1-3章：入局与身份建立。
- 4-6章：第一次小胜 + 暴露更高风险。
- 7-9章：敌人/规则开始反击。
- 第10章：大冲突闭环 + 推入下一单元。

前十章每章结束后，读者必须能说出：

- 主角这一章赚到了什么？
- 主角这一章失去了什么？
- 下一章必须马上干什么？

### 7.3 长篇扩容

长篇不能只靠前十章吸引。

必须持续扩容：

- 个人危机 → 组织危机 → 城市异常 → 系统底层 → 终局选择
- 钱 → 命 → 记忆 → 关系 → 身份 → 现实稳定
- 低级 bug → 人物 bug → 合同 bug → 记忆 bug → 系统级 bug → 林川自身 bug

“账”不能局限为金钱概念。它是祸福相依的机缘：一边给寿命、现金、证据、权限和现实修复机会，一边收走记忆、关系、身份、公共记录、他人命运和现实稳定。后续高武化外显必须从这套“账”里长出来，不另起一个练级系统。

每 10 章检查一次主问题是否切换；每 30 章检查一次舞台是否扩大。

## 8. 系统/规则架构标准

现实 bug 系统不是外挂菜单，而是小说发动机。

必须逐步建立十层：

1. 检测层：林川能看到现实错误。
2. 复现层：必须找到 bug 发生条件。
3. 修复层：可临时补丁、彻底修复、拒绝修复、错误修复。
4. 收益层：寿命、现金、记忆碎片、权限、证据、关系修复窗口。收益要像机缘，但必须带刺；“账”给主角往前走的机会，也必须同步标价。
5. 代价层：记忆丢失、关系清除、证据消失、他人命运改写、寿命扣减。前几次代价要能回读识别：首读有刺，后文回看时恍然，不写成说明书。
6. 回滚层：修错会把现实回滚到更坏版本。
7. 污染层：bug 扩散到人、合同、记忆、空间、城市流程。
8. 权限层：林川最初只是低权限修复者，后期发现管理员权限。
9. 敌方层：刘成/回收进程不是单纯追杀，而是在执行异常清理。
10. 终局层：林川自己可能是第一个 bug、最大容器或被系统漏删的人。

每章至少推进其中一层。

系统提示必须像判决书、账单、死亡通知或错误日志，不能像说明书。

示例方向：

```text
【修复完成】
```

## 10. 人物工程标准

### 10.3 刘成/刘哥

## 11. 商业留存标准

每章必须考虑五个转化：

1. 点开：标题、简介、前 300 字是否有强问题。
2. 留存：读者是否在前 1-3 章知道这本书退不了。
3. 追读：章末是否让读者停不下。
4. 收藏：人物、设定和长期谜题是否值得陪伴。
5. 评论：是否有站队、骂点、心疼点、选择困境或可讨论句。

每章要设计至少一个评论触发点，例如：

- 修 bug 该不该牺牲别人的记忆？
- 林川该不该告诉苏念孩子存在？
- 刘成到底是敌人还是执行规则的人？
- 如果救一个人会删除另一个人，应该修吗？
- 王海被删前到底背叛了谁？

### 11.1 社会热梗和现实痛点评论爆点

新增执行口径：本书可以使用社会热梗、现实痛点和当代情绪作为评论触发器，但必须长在人物处境和剧情冲突里，不能像短视频文案、段子合集或作者夹带观点。

优先使用“稳定社会情绪”，少用寿命很短的网络流行词。适合本书的评论爆点包括：

- 欠债、催收、体面社死、成年人不敢崩溃。
- 打工人/牛马感、项目背锅、公司倒闭后个人还债。
- 医院建档、父亲字段、孩子身份、普通人办事被系统字段卡住。
- 合同不让验货、平台自动扣款、先签字后解释、普通人被流程吞掉。
- 为了亲人继续干脏活、底层人互相为难、老马这种“坏事做过但不是坏人”的灰度。
- “不是不努力，是账会算利息”这类可截图句，但必须由现场自然推出来。

禁用方式：

- 硬塞当下热词、谐音梗、互联网烂梗。
- 让角色突然像评论区嘴替或短视频博主。
- 用金句总结替代剧情行动。
- 为了评论点牺牲悬疑压迫感或人物真实感。
- 每章都用同一种“成年人太难了”口号，造成疲劳。

执行标准：每 1-2 章至少有一个可引发评论的现实刺点；评论点必须绑定人物选择、现实制度、亲情/关系代价或账单规则。读者应当想评论“这不就是现实吗”“这单不能签”“老马这种人太真实了”“字段两个字太吓人”，而不是觉得作者在硬蹭热梗。

控制爽虐比例：高压之后必须给小胜利、小温情、小反击或信息兑现，不能一直压迫到读者疲劳弃读。

## 12. 章末钩子标准

章末不能下课，不能总结本章，不能预告下一章。

可用钩子类型：

1. 新证据型：账不对、监控不对、照片不对、人也不对。
2. 新敌人型：刘成、回收进程、新组织、新权限者入场。
3. 胜利反噬型：刚修好，现实倒退到更坏版本。
4. 误解/背锅型：林川好心修复，却被现实记录成凶手。
5. 时限压迫型：今晚必须行动，明天债务/寿命/记忆到期。
6. 关系刺点型：苏念下意识记得一个不该记得的动作。
7. 选择困境型：修复她，删除他；保留记忆，扣掉寿命。
8. 身份暴露型：系统日志显示林川不是修复者，而是容器。

章末钩子必须戳到人物在乎的东西，不能只是“又出现新 bug”。

### 18.4 现实 bug 系统锁定细则

检测层：林川能看到时间异常、空间异常、人物异常、合同异常和物品异常。初期只能看到低级 bug，中期解锁人物 bug，后期解锁系统级 bug。主动扫描会消耗精力，过度使用会头痛、虚弱或流鼻血。

复现层：修复前必须找到触发条件，否则只能临时压制。条件可以是时间、空间、人物、行为或组合条件。复现可能加剧扩散、触发回收进程注意或让更多人卷入异常。

修复层：修复模式包括临时补丁、彻底修复、拒绝修复、错误修复。修复必须在 bug 扩散到临界点前完成，否则只能回滚或隔离。

收益层：收益可以是寿命、现金、记忆碎片、权限、证据、关系修复窗口、敌方漏洞或新目标。收益不能白给，必须绑定扣除项、暴露风险、关系代价或回收端反制。

代价层：代价包括记忆丢失、关系清除、证据消失、他人命运改写、寿命扣减和精力消耗。代价与收益成正比，不可预测且不可逆。

回滚层：错误修复、顺序错误、时机错误或权限不足会导致现实回滚到更坏版本，并可能触发更严重 bug 或回收进程强制介入。

污染层：bug 不修复会持续扩散，路径为人 → 物品 → 空间 → 时间 → 规则。扩散到临界点后无法清除，只能回滚或隔离。

权限层：林川初期是低权限修复者，中期可修人物 bug，后期可修系统级 bug；终局要发现自己可能不是修复者，而是接口、容器或漏删对象。

敌方层：刘成不是普通催债人，而是回收进程执行者；回收进程不是人，是系统自动运行的清理程序，表现形式可为合同、电话、短信、系统提示、刘成或 R-0512。

前十章已锁定规则：关系债目冻结、记忆回流账单、公开记录归零 vs 体内残余节点、空间折叠与 0512 回收车、违规备份链、记忆分层、黑盒子、拒单、容器、候选接口与七日异常复核期。
